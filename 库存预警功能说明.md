# 库存预警功能使用说明

## 功能概述

库存预警功能允许管理员设置一个库存阈值，当商品的库存数量低于该阈值时，商家在后台管理界面的商品列表中会看到该商品的库存预警提示。

## 功能特点

1. **管理员可配置阈值**：管理员可以根据实际需求设置库存预警阈值
2. **实时预警提示**：商家在商品管理页面可以直观地看到哪些商品需要补货
3. **权限分离**：只有管理员可以设置阈值，所有商家用户可以看到预警效果

## 后端实现

### 1. 新增实体类

- `SysConfig.java` - 系统配置实体类，用于存储库存预警阈值等系统参数

### 2. 新增 Mapper

- `SysConfigMapper.java` - 系统配置数据访问层

### 3. 新增 Service

- `ISysConfigService.java` - 系统配置服务接口
- `SysConfigServiceImpl.java` - 系统配置服务实现类

### 4. 新增 Controller

- `InventoryWarningController.java` - 库存预警控制器
  - `GET /inventory/threshold` - 获取当前库存预警阈值
  - `POST /inventory/threshold` - 设置库存预警阈值
  - `GET /inventory/warning/list` - 获取需要预警的商品列表
  - `GET /inventory/warning/count` - 获取库存预警统计信息

### 5. 修改现有文件

- `Goods.java` - 添加 `needWarning` 字段（仅用于前端展示）
- `GoodsController.java` - 在 `findPage` 方法中为商品添加预警标识
- `AdminController.java` - 添加 `setThreshold` 方法供管理员设置阈值

### 6. 数据库变更

执行 `src/main/resources/inventory_warning.sql` 创建系统配置表并初始化默认值

## 前端实现

### 1. 新增页面

- `InventoryWarning.vue` - 库存预警设置页面（仅管理员可见）
  - 显示当前预警阈值
  - 提供设置阈值的对话框

### 2. 修改现有页面

- `Goods.vue` - 商家后台商品管理页面
  - 添加加载阈值的功能
  - 在库存数量列显示预警标签（红色高亮）

### 3. 路由配置

- 在 `router/index.js` 中添加库存预警设置页面路由
  - 路径：`/back/inventory-warning`
  - 权限：仅管理员可访问

### 4. 菜单配置

- 在 `Back.vue` 中添加"库存预警设置"菜单项
  - 位置：商品管理下方
  - 图标：WarningFilled
  - 权限：仅管理员可见

## 使用流程

### 管理员操作

1. 登录后台管理系统
2. 点击左侧菜单"库存预警设置"
3. 查看当前预警阈值
4. 点击"配置阈值"按钮
5. 输入新的阈值（0-1000）
6. 点击"确定"保存

### 商家查看

1. 登录后台管理系统
2. 点击左侧菜单"商品管理"
3. 在商品列表中查看"库存数量"列
4. 库存低于阈值的商品会显示红色预警标签，格式为："XX (预警)"
5. 库存正常的商品只显示数字

## API 接口说明

### 1. 获取库存预警阈值

**请求：**
```
GET /inventory/threshold
```

**响应：**
```json
{
  "code": "200",
  "data": 10
}
```

### 2. 设置库存预警阈值（管理员）

**请求：**
```
POST /inventory/threshold
Content-Type: application/json

{
  "threshold": 10
}
```

**响应：**
```json
{
  "code": "200",
  "msg": "操作成功"
}
```

### 3. 获取需要预警的商品列表

**请求：**
```
GET /inventory/warning/list?pageNum=1&pageSize=10&keyword=
```

**响应：**
```json
{
  "code": "200",
  "data": {
    "records": [
      {
        "id": 1,
        "name": "商品名称",
        "inventory": 5,
        "needWarning": true,
        ...
      }
    ],
    "total": 1
  }
}
```

### 4. 获取库存预警统计

**请求：**
```
GET /inventory/warning/count
```

**响应：**
```json
{
  "code": "200",
  "data": 5
}
```

## 注意事项

1. 首次使用前需要先执行 SQL 脚本创建系统配置表
2. 默认阈值为 10
3. 阈值必须为非负整数
4. 管理员和商家都可以查看预警商品列表
5. 只有管理员可以修改阈值
6. 商品管理页面的预警标识是实时的，每次刷新都会重新计算

## 技术要点

1. 使用 `@TableField(exist = false)` 注解标记不存储到数据库的字段
2. 通过 TokenUtils 获取当前登录用户信息进行权限控制
3. 使用 LambdaQueryWrapper 构建类型安全的查询条件
4. 前端使用 Element Plus 的 Tag 组件显示预警标识
5. 采用统一的 Result 返回格式
